Exemple d'une application de lecture d'articles avec une zone administration pour la gestion des données.
L'auteur d'un nouvel article devra saisir les informations nécessaires dans le formulaire.
L'article possédera automatiquement un numéro unique, une date de publication, et se positionnera dans les différentes sessions de l'application. Le but est de permettre aux administrateurs de saisir rapidement et simplement un article sans connaissances techniques. <> Code Développé en Angular, pensée "mobile first" type application mobile. Les nouveaux articles seront auto-incrémentés dans les composants avec une boucle "*ngFor", le crud se fera sur une api pour la gestion des données avec un espace connexion protègé "guard".
Dans le cas présent, le code sera injecté en dur dans le service et réinitialisé à chaque rafraichissement.
L'application est minimaliste dans un simple but démonstratif.
