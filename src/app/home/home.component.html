Exemple d'une application de lecture d'articles avec une zone administration pour la gestion des données.
L'auteur d'un nouvel article devra saisir les informations nécessaires dans le formulaire.
L'article possédera automatiquement un numéro unique, une date de publication, et se positionnera dans les différentes sessions de l'application. Le but est de permettre à l'administrateur de saisir rapidement et simplement un article sans connaissances techniques. ★ Code Développé en Angular, le but de cette application est de permettre de lire des articles de manière simple et efficace, sur téléphone "mobile first" ou ordinateur. Les nouveaux articles sont auto-incrémentés dans le code une fois créés et envoyer vers l'api pour le stockage dans la bdd.
Dans le cas présent, le code sera injecté en dur dans le service et réinitialisé à chaque rafraichissement.
Les articles sont minimalistes dans un simple but démonstratif.
